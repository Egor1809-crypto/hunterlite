## Заметки по текущему состоянию

### 2026-05-16. Auth и защита входа

Добавили backend-авторизацию с cookie-сессией и защиту от частого перебора пароля на `/api/auth/login`.
Инструменты блока: `apps/api/src/modules/auth`, HTTP resolver, Vitest, ESLint, Vite build.
Проверка: `npm run test`, `npm run lint`, `npm run build`.

### 2026-05-16. CSRF-защита

Добавили CSRF-cookie при логине и проверку `X-CSRF-Token` для POST-запросов с активной сессией.
Инструменты блока: `apps/api/src/modules/auth/csrf.ts`, `src/lib/api-client.ts`, HTTP resolver, Vitest.
Проверка: `npm run test`, `npm run lint`, `npm run build`.

### 2026-05-16. Временная блокировка аккаунта

Добавили хранение количества неверных входов и `locked_until` в `auth_accounts`. После серии неверных паролей аккаунт временно не создает новую сессию, а успешный вход сбрасывает счетчик.
Инструменты блока: Prisma schema/migration, `createAuthPrismaDataSource`, seed, Vitest.
Проверка: `npm run test`, `npm run lint`, `npx prisma validate`, `npm run build`.

### 2026-05-16. Восстановление пароля

Добавили backend-сценарий восстановления пароля: запрос reset-токена, хранение только hash токена, срок действия, одноразовое использование и смену password hash.
Инструменты блока: Prisma schema/migration, `createAuthPrismaDataSource`, auth HTTP routes, `src/lib/frontend-api.ts`, `Login.tsx`, Vitest.
Проверка: `npm run test`, `npm run lint`, `npx prisma validate`, `npm run build`.

### 2026-05-16. Роли и права доступа

Добавили общий permission-контракт для API-действий, backend-проверку прав по permission, а frontend-навигацию привязали к `canAccessRoute`.
Инструменты блока: `src/lib/access-control.ts`, HTTP resolver, `AppSidebar`, Vitest.
Проверка: `npm run test`, `npm run lint`, `npm run build`.

### 2026-05-16. Перенос данных на API

Основные страницы уже используют `frontendApi`; backend dev server подключает Prisma-backed data source. Профиль теперь собирает пользователя из Prisma users и слабые темы из Prisma weak topics, без прямой склейки с demo profile fallback.
Инструменты блока: `createBackendDataSource`, Prisma data sources, `frontendApi`, React Query fallback.
Проверка: `npm run test`, `npm run lint`, `npm run build`.

### 2026-05-16. API-состояния на страницах

Добавили общий `ApiState` и подключили loading/error/empty состояния на основные API-страницы: dashboard, профиль, история, слабые темы, уведомления, manager overview и настройка сессии.
Инструменты блока: React Query, `ApiState`, `useApiData`.
Проверка: `npm run test`, `npm run lint`, `npm run build`.

### 2026-05-16. Блок 7: правила тренировок и экзаменов

Зафиксировали MVP-логику: 3 режима, баллы 0-100, проходной балл 70, критерии оценки, разные персонажи AI-клиента, лимит вопросов по сложности до 100 и sequence-тесты по процедурам банкротства.
Инструменты блока: `src/lib/training-logic.ts`, `docs/BLOCK_7_TRAINING_AI_LOGIC.md`, Vitest.
Проверка: `npm run test`, `npm run lint`, `npm run build`.

### 2026-05-16. Backend API тренировочных сессий

Добавили создание training session, сохранение сообщений и завершение сессии с результатом. В Prisma добавлены поля формата, персонажа, количества вопросов, критериев, ошибок и рекомендаций.
Инструменты блока: `TrainingSession`, `createTrainingsPrismaDataSource`, HTTP resolver, permission policies, Prisma migration.
Проверка: `npm run test`, `npm run lint`, `npx prisma validate`, `npm run build`.

### 2026-05-16. Frontend-подключение training session API

Старт тренировки теперь создает backend-сессию и передает `sessionId` в экран диалога. Сообщения пользователя/AI сохраняются через API, а завершение сессии отправляет балл, критерии, ошибки и рекомендации.
Инструменты блока: `SessionSetup`, `SessionChat`, `SessionResult`, `frontendApi`, Prisma datasource tests.
Проверка: `npm run test`, `npm run lint`, `npx prisma validate`, `npm run build`.

### 2026-05-18. Интеграция ТЗ в БД
Добавили модели данных для всех 4 модулей проверки знаний из ТЗ (Тесты, Ситуационные кейсы, Отработка возражений, Конструктор симуляции звонков), а также TrainingProgram для сборки курсов.
Инструменты блока: Prisma schema, Prisma client.
Проверка: `npx prisma validate`, `npx prisma generate` выполнены. Требуется `npx prisma migrate dev`, когда будет запущен Docker.

### 2026-05-20. Локальная платформа и API-контракт
Подняли локальный PostgreSQL, применили существующие миграции, прогнали seed и проверили реальный login API демо-пользователя. Закрыли базовый цикл сотрудника: вход, согласие, dashboard, создание training session, диалог, результат и запись в истории.
Дополнительно синхронизировали frontend/backend paths для admin content API: `/api/admin/tests`, `/api/admin/cases`, `/api/admin/objections`, `/api/admin/call-scripts`; добавили employee-readable `/api/trainings/call-scripts`; закрыли admin content endpoints permission policies.
Проверка: `npm run test` — 205 тестов прошли.

- Основные данные сейчас моковые и лежат во frontend.
