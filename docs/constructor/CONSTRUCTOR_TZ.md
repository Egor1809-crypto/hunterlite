## 0. Контекст и цель

Платформа — обучение юристов по **банкротству ФИЗИЧЕСКИХ лиц (ФЗ-127)**. Панель
`/training` имеет две вкладки:

- **«Тесты»** = `TestWorldMap` (10 регионов × 10 уровней = острова знаний). **Это
  продукт, НЕ трогаем** (кроме чтения прогресса для разблокировки, см. §4).
- **«Мои клиенты»** = `CharacterBuilder` (конструктор персонажа). **Это предмет
  данного ТЗ.**

**Цель конструктора:** дать юристу отрабатывать навыки разговора с клиентом-должником
(физлицо) в формате **чата или звонка** — применять закон и опыт на практике.
Режим **свободный и необязательный**: конструктор НЕ является обязательным шагом
перед экзаменом.

**Решения заказчика (зафиксированы):**
1. Пресеты («Обучающие пресеты») — **удалить полностью**. Оставить только
   **Свободный конструктор**. Сделать **один эталонный персонаж-физлицо**, на основе
   которого позже масштабируем (через `/ultracode`).
2. **Удалить все «уровни»** (XP-гейтинг шагов). Разблокировку привязать к
   **прохождению островов теста**.
3. **Один звонок.** Режим «AI×N» (мульти-звонок story) и вся CRM-обвязка
   реального клиента — старый мусор, убрать с пути конструктора.
4. Из рекомендованного убрать всё, связанное с банкротством **юридических** лиц.

## 2. Эталонный персонаж-физлицо (единый образец качества)

`reference_personas`, чтобы не смешивать с пользовательскими сохранениями).

**Спека эталона (все поля заполнены реально, ФЗ-127, физлицо):**

## 3. Разблокировка по островам теста (новое правило)

Удаляем XP-гейтинг (`CharacterBuilder.tsx:88-95` `unlockLevel`, `:234-236`
`useGamificationStore.level`, `isStepLocked`). Вместо этого:

**Правило (DECISION-4, рекомендуемый дефолт):**
- Конструктор («Мои клиенты») **заблокирован**, пока не пройден **первый регион
  острова** (регион 1 «Условия подачи» = 10 уровней, каждый ≥ `PASS_THRESHOLD`).
- После этого — **полностью свободен**: все 8 шагов открыты, без доп. гейтов,
  необязателен, не блокирует экзамен.
- Источник истины: `GET /training-map/progress` → `test_map`; вывести
  `constructor_unlocked: bool` (бэкенд-derived или FE-расчёт по `test_map`).

> Альтернативы для DECISION-4: (а) разблокировка после **1 уровня** (минимальный
> гейт); (б) **без гейта вообще** (конструктор всегда свободен, «привязка к островам»
> = только рекомендация сложности). Заказчик выбирает.

Контракт: добавить в ответ `/training-map/progress` поле
`constructor_unlocked` + (для UI) `constructor_unlock_hint` («Пройдите регион
"Условия подачи", чтобы открыть практику»).

## 4. Бэкенд-контракт после чистки

**Удаляются:** `training_presets` (модель/таблица/роутер/seed). 
**Остаются:** `custom_characters`, `training_map_progress`, `POST /training/sessions`.

`POST /training/sessions` — `SessionStartRequest` для конструктора принимает только:
`custom_character_id?`, плоские `custom_*` (archetype/profession/lead_source/
difficulty/family/creditors/property/emotion/environment/fatigue/tone), `mode ∈
{chat, call}`. **Запрещены/игнорируются** story/CRM-поля (§1.2). Гард старта
(`evaluate_start_guards`) сохраняется (энергия/лимиты), но без story-веток.

Ошибки (обязательная обработка):
- конструктор заблокирован (регион не пройден) → `403 {code: "constructor_locked"}`.
- нет энергии → существующий guard, `409`/`{code}` как сейчас.
- невалидный `custom_*` (пустой архетип и т.п.) → `422` с полем.
- LLM недоступен на старте досье → деградация: стартуем сессию без `cached_dossier`,
  лог `warning`, не падаем.

## 5. Фронт (`CharacterBuilder` после чистки)

### 5.1 Структура
- Вход сразу в **wizard** (нет экрана `choice` с двумя картами — пресетов больше нет).
- 8 шагов, **все доступны** (если конструктор разблокирован). Если заблокирован —
  экран-заглушка с `constructor_unlock_hint` и кнопкой «Перейти к тестам».
- Финал: **две** кнопки — «Чат» и «Звонок» (без «AI×N»).
- Чекбокс «Сохранить в Мои клиенты» — остаётся.

### 5.2 Вкладка/лейбл
- Вкладка называется «Мои клиенты» (как сейчас). Текст и копирайт — в стиле lendobserv
  (см. фронт-ТЗ): без игрового жаргона.

### 5.3 Критерии оценки разговора (rubric эталона)
Сессия оценивается по under-the-hood метрикам (используются существующие
`score_*`): script_adherence, objection_handling, communication, result. Для
эталона задать целевые пороги «хорошего разговора» (выяснил состав долга/имущество/
иждивенцев; корректно объяснил применимость реструктуризации vs реализации; не
спутал с банкротством юрлиц). Эти же метрики идут в `/history` (см. фронт-ТЗ).

## 6. Edge cases / обработка ошибок
- Удаление `training_presets` не должно ломать `seed_all.py` — убрать пресеты из
  оркестратора; добавить `seed_reference_persona`.
- `app/models/__init__.py` `import *` (путь alembic) — после удаления `TrainingPreset`
  убедиться, что нет висячего экспорта (иначе collection-error, как было в TZ-1).
- Миграция `drop table training_presets` — обернуть в `sa.text`, проверить
  `alembic upgrade head` локально/в CI (advisory, но schema-drift не должен ругаться).
- FE: после удаления `PresetRecommendation`/preset-ветки — `tsc --noEmit` чисто,
  нет мёртвых импортов (см. урок TZ-1).

## 7. Критерии готовности (Definition of Done)
1. Пресет-подсистема удалена целиком (модель/таблица/роутер/seed/FE), `app.main`
   импортируется, `from app.models import *` ОК, миграция drop проходит `upgrade head`.
2. Конструктор открывается без пресетов; 8 шагов без XP-локов; разблокировка по
   правилу §3; заблокированное состояние показывает hint.
3. Только «Чат»/«Звонок»; «AI×N»/story-URL отсутствуют; путь конструктора не шлёт
   story/CRM-поля; старт сессии работает (браузерная проверка: создать персонажа →
   запустить чат и звонок).
4. Один эталонный персонаж-физлицо засеян и проходит юридическую проверку (ФЗ-127,
   без юрлиц).
5. Из рекомендованного/каталога убрано всё юрлицо.
6. `tsc --noEmit` чисто; CI blocking (TZ-1 scope) зелёный; пуш по §1.
