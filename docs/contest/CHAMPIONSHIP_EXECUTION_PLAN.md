## Прогресс реализации (2026-06-10)
- ✅ **Фаза A** — коммит ждёт команды; публичность Положения, ссылка в футере, WebP (−96%), `/championship` везде. Попутно починен баг bounce гостей на `/login`.
- ✅ **Фаза B** — таб «Чемпионат» на лендинге, пункт меню Trophy, Маняша на лендинге, hero-эффект таяния.
- ✅ **Фаза C** — джоб квалификации, гонка-гард enroll (+`asyncio.gather`-тест), FSM сезона (lazy-on-read + scheduler), операторский endpoint розыгрыша. **11/11 тестов зелёные.**
- ✅ **Фаза D** — Положение §2.1 приведено к реализации (аттестация + платная подписка с курсами + отзыв); §1.3 «не лотерея/участие бесплатно» и §4.1 «жребий среди квалифицированных, не по рейтингу» уже корректны; знания чемпионата добавлены в системный промпт Маняши (`/api/chat`); НДФЛ-дисклеймер на карточках призов. Юр-виза (D.4) — внешний шаг, дисклеймер-баннер на странице Положения.
- ✅ **Фаза E** — admin-модерация отзывов (`/reviews/pending`, `/approve`, `/reject`) + тест (**12/12**); вау-эффект призов (scroll-driven 3D-наклон+зум на одиночном изображении, reduced-motion gated); адаптив+тёмная проверены (нет оверфлоу на 375px, корректный тёмный фон/текст).
- ⏳ **Отложено (после всех фаз):** реальный трекинг прохождения курсов; колонка `draw_proof` для юр-аудита; **полное 360° frame-вращение призов** (нужен набор кадров на девайс — asset-задача, сейчас честный single-image 3D-эффект); коммит A–E.

### Новые белые пятна, найденные в Фазе C
- 🔵 **«Курсы» нигде не трекаются** — нет моделей course/lesson/progress. Решено: квалификация-гейт = `аттестация(сертификат ≥88%) AND активная платная подписка AND отзыв`; `courses_done` = прокси по подписке (платные планы включают оба курса). Требует подтверждения: ок ли так, или ждём реального трекинга курсов.
- 🔵 **Пруф розыгрыша не персистится** — `random_org_verification` сейчас только логируется (нет колонки). Для юр-аудита стоит добавить `draw_proof` JSONB в `championships` (мини-миграция, Фаза E) — фиксация результата хранится у random.org + видео, но ссылку лучше хранить и у нас.
- ℹ️ **Розыгрыш = внешняя случайность.** Endpoint НЕ генерирует случайность сам (это подорвало бы юр-фиксацию) — админ проводит жребий на random.org и POST'ит результат (rank→user + ссылка). `auto_rank=true` — fallback по баллам для `winner_mode='ranking'`.

## 0. TL;DR

Готов качественный **UI-каркас + read-only API + текст Положения**. Отсутствует вся
**живая механика** (квалификация, FSM сезона, розыгрыш), **Положение недоступно публично и
ни с чем не связано**, **Маняша не знает правил**, **ничего не закоммичено**. Дизайн страницы
по факту уже на 80% совпадает с Apple-паттернами из ресёрча — апгрейд точечный, не переписывание.

## 2. Логические ошибки и «белые пятна» (как просил — выписать)

1. **Противоречие «жребий vs объективные критерии».** Базовый план §7 п.2 говорит «победитель — по
   объективным критериям, **не жребий**», а §0 решение — **рандомайзер**. Текст Положения и механика
   расходятся. **Фикс формулировки:** квалификация = объективный гейт (допуск), розыгрыш = жребий
   среди допущенных. Положение должно говорить именно так, без «победитель по баллам».
2. **«Де-факто платное участие».** В ТЗ есть фраза «участие платное, т.к. нужны платные курсы» —
   юридически это читается как скрытая лотерея/плата за билет. **Правильная рамка:** участие
   **бесплатно**; платные курсы — часть продукта, а не взнос. Переформулировать в Положении и UI.
3. **Даты сезонов размыты.** «Последняя неделя весны/осени» — не календарная константа.
   Весна кончается 31 мая, осень — 30 ноября. Нужны **точные формулы** старта/tally/финиша на сезон,
   иначе FSM не построить. Предлагаю якорить так (см. §5.2). Сейчас сид хардкодит только сезон №1
   (tally 2026-11-24, end 2026-11-30) — для FSM этого мало.
4. **Условие «оставить отзыв» — мёртвое.** `review_left` считается, но `qualified` нигде не
   выставляется → отзыв ни на что не влияет. Связать через джоб квалификации (R4).
5. **Маняша обещает, но не знает.** Интро на странице говорит «спросите про условия/НДФЛ/сроки», а
   знаний в промпте нет → будет галлюцинировать. Либо дать знания (R10), либо убрать обещание.
6. **НДФЛ-агент.** Призы неденежные → организатор удержать налог не может, победитель сам
   декларирует (3-НДФЛ). В UI и Положении обязательно явно (есть в тексте; продублировать на карточке
   приза, чтобы не было споров).
7. **Согласие ст.10.1 152-ФЗ** на публикацию имени/фото — отдельный чекбокс при вручении, не
   смешивать с общим согласием на обработку ПДн.
8. **Нумерация сезонов.** «Сквозной №» требует, чтобы FSM **создавал следующий** чемпионат, а не
   только переключал статус текущего. Сейчас только сид создаёт №1.
9. **Пьедестал убираем — ранги оставляем.** Ранг = порядок вытягивания (1→MacBook, 2→iPhone,
   3→AirPods). Графику-пьедестал не рендерим (уже учтено в `RANK_LABEL`).

## 3. Дизайн-система из ресёрча (что подтвердилось, что апгрейдим)

> Вывод ресёрча: текущая страница **уже использует** ядро Apple-паттернов (useScroll/useTransform,
> sticky-призы, scaleX-прогресс, clamp-типографика, один акцент, reduced-motion). Апгрейд —
> **точечный**, не переписывание. Высокая уверенность — первичные доки motion.dev/web.dev/MDN/Apple.

### 3.1 Подтверждённые техники (high-confidence, можно брать as-is)
| Техника | Как | Статус у нас |
|---|---|---|
| Прогресс-бар сверху | `<motion.div style={{scaleX: scrollYProgress, originX:0}}/>`, опц. `useSpring` | ✅ есть (`:67,:289`) |
| Reveal/blur/parallax | `useTransform(p,[0,1],['blur(10px)','blur(0px)'])`, opacity, y | ✅ есть |
| Per-element pin призов | `useScroll({target:ref, offset:['start end','end start']})` | ✅ есть (`PrizeStory`) |
| Sticky-горизонталь (опц.) | tall-контейнер + `position:sticky` + `x=useTransform(...,['0%','-75%'])` | ⛔ не используем (не нужно) |
| Только `transform`/`opacity` (GPU, бюджет 16.7ms) | не анимировать layout/paint | ✅ соблюдаем |
| `prefers-reduced-motion` (opt-in под no-preference) | `useReducedMotion()` | ✅ есть |
| SF Pro системный стек | `-apple-system, BlinkMacSystemFont, "SF Pro"` | проверить в globals.css |

| Один акцент-pill CTA | наш бренд-акцент (НЕ `#0071e3` — он только структурный ориентир) | ✅ malvah: monochrome+1 акцент |

### 3.2 Апгрейды (то, чего нет — приоритезировано)
- **A1 (must, дёшево). Оптимизация фото призов в WebP.** Сейчас .png 1.3/1.1/0.9 МБ — ресёрч:
  WebP режет ~90%. 3 файла → ~150-400 КБ суммарно. Лендинг-страница «самая продажная» → вес критичен.
- **A2 (should). Hero-типографика тает/масштабируется при скролле** (`scale 1→0.92` + opacity) —
  явно в ТЗ «эффект с самого начала». Проверить, есть ли уже; если нет — добавить.
- **A3 (could, тяжёлое, ФАЗА 4). Canvas image-sequence** — «фирменное» вращение девайса (Apple-приём:
  пронумерованные кадры → `drawImage` на canvas по scrollTop, GSAP ScrollTrigger `scrub/pin/ease:none`).
  Тяжело по ассетам (нужен набор кадров на девайс) → **откладываем**, sticky-reveal сейчас достаточно.
- **A4 (should). Маняша на лендинг-странице** — смонтировать виджет прямо в секции (лендинг без
  AuthLayout). Уже есть на `/championship` route (`:53`) — проверить, чтобы и в лендинг-вкладке был.

### 3.3 Структура страницы (funnel — reasoned default, не из CRO-исследования)
hero (заголовок + countdown) → showcase призов (sticky-reveal, 1 экран/приз) → как участвовать
(3-4 шага, stagger reveal) → пул квалифицированных / розыгрыш (ранги без пьедестала) →
история победителей (empty-state) → отзывы (social proof) → футер с документами/ТГ-ботом.
> Порядок — разумный дефолт; A/B-данных по sweepstakes ресёрч не подтвердил (openQuestion).
