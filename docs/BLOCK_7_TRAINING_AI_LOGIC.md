# Блок 7. Тренировки, экзамены и AI-логика

Цель блока: превратить тренировку из набора экранов в backend-сценарий, где сессия создается, проходит по правилам, сохраняет сообщения, ответы, результат, ошибки и рекомендации.

## Режимы первой версии

- `talk` — тренировка с AI-клиентом.
- `exam` — экзамен с проходным баллом.
- `chat_test` — тестовый формат, включая задания на правильную последовательность шагов.

## Оценка

- Балл: от 0 до 100.
- Проходной балл: 70.
- Критерии:
  - юридическая точность;
  - структура ответа;
  - безопасные формулировки;
  - эмпатия;
  - закрытие возражения.

## Сложность и количество вопросов

- Базовая: до 20 вопросов.
- Средняя: до 50 вопросов.
- Сложная: до 100 вопросов.

## Что сохраняем после сессии

- Тема.
- Режим.
- Сложность.
- Формат.
- Персонаж клиента.
- Сообщения пользователя и AI.
- Итоговый балл.
- Ошибки.
- Рекомендации.
- Статус: черновик, завершено, не сдано, сдано.

## AI-клиент

Персонаж клиента должен меняться от сессии к сессии:

- тревожный;
- агрессивный;
- сомневающийся;
- недоверчивый;
- торопящийся.

## Sequence-тесты

Отдельный тип задания: юрист получает перемешанные этапы процедуры банкротства и должен расставить их в правильном порядке.

Пример: сбор документов, подача заявления, введение процедуры, решение суда.

## Кодовый контракт

- `src/lib/training-logic.ts` — режимы, критерии, сложности, проходной балл, topics и проверка sequence-задач.
- `src/test/training-logic.test.ts` — unit-тесты правил блока.

## Backend API

- `POST /api/trainings/sessions` — создать тренировочную/экзаменационную сессию.
- `POST /api/trainings/sessions/:id/messages` — сохранить сообщение пользователя или AI.
- `POST /api/trainings/sessions/:id/complete` — завершить сессию, сохранить балл, критерии, ошибки и рекомендации.

Права доступа: только пользователь с permission `training:create:self` / `training:complete:self`.
