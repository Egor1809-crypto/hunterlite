# HunterLite — платформа обучения арбитражных управляющих

Онлайн-обучение специалистов по банкротству физических лиц (ФЗ-127) через
AI-симуляции: живой диалог с клиентом-должником, тесты по закону, экзамены с
проверкой ИИ и **диплом/сертификат** по итогам аттестации.

Продукт в интерфейсе называется **LegalHunter**. Прод: **https://legalhunter.pro**.

> **Это реальное обучение, а не игра.** Платформа выросла из кода геймифицированного
> тренажёра, но переосмыслена в серьёзную обучающую систему с аттестацией и
> сертификацией. Игровой/соревновательный слой (PvP, турниры, лидерборды) вырезан.
> Если встречаете в коде остатки «арены» — см. раздел «Наследие» ниже.

## Что внутри

| Раздел | Что делает |
|---|---|
| **Тренировка** | AI-**чат** и AI-**звонок** с клиентом-должником. Диалог на русском, разбор по итогам. |
| **Тесты** | Карта из ~1500 вопросов по ФЗ-127, разбитых на тематические блоки («острова»). |
| **Экзамен / Сертификат** | 5 экзаменов, 6 механик (выбор, последовательность, разбор дела, составление документа и др.), проверка ответов ИИ. Сертификат с публичной верификацией по коду. |
| **База знаний** | Браузер по ФЗ-127 + AI-ассистент **«Маняша»** (RAG по 624 выверенным фрагментам закона и практики). |
| **Кейсы БФЛ** | 50 интерактивных кейсов: дерево решений + восстановление хронологии дела. |
| **Радар** | Лента изменений законодательства и судебной практики. |
| **Курсы** | Видеокурсы с дрип-подачей и мини-проверками. |
| **Чемпионат** | Периодический конкурс среди учеников (розыгрыш призов, юридически оформленный). |
| **Dashboard / KPI** | Аналитика для руководителей (РОП): прогресс команды, слабые темы. |

## Технологический стек

- **Frontend** (`apps/web`) — Next.js 15 (App Router), React 19, TypeScript, Tailwind,
  Zustand, Framer Motion.
- **Backend** (`apps/api`) — FastAPI, async SQLAlchemy 2, Alembic.
- **Хранилища** — PostgreSQL + **pgvector** (RAG-эмбеддинги), Redis.
- **ИИ** — весь LLM / STT (распознавание речи) / TTS (синтез) идёт через **один провайдер
  navy.api** (OpenAI-совместимый прокси). Прямые Gemini/OpenAI/Claude из кода убраны.
  Основная модель — `deepseek-v4-pro`; быстрая для живого диалога — `claude-haiku-4-5`.
- **Telegram-бот** — `@BFLHUNTER_bot`, работает в режиме long-polling.

## Запуск локально

Требуется Node.js 20+, Python 3.13, PostgreSQL (с pgvector), Redis. Ключи ИИ (`NAVY_LLM_*`)
кладутся в `apps/api/.env` (gitignored).

```bash
make dev        # поднять всё через Docker (web + api + db + redis)
make migrate    # alembic upgrade head
make seed       # базовые dev-данные
make test       # тесты обеих сторон
make lint       # линт
```

Раздельно:

```bash
cd apps/web && npm install && npm run dev     # http://localhost:3000
cd apps/api && uvicorn app.main:app --reload  # http://localhost:8000
```

Полная загрузка контента (тесты, кейсы, экзамены, база знаний, чемпионат):

```bash
docker compose ... exec api python -m scripts.seed_all
docker compose ... exec api python -m scripts.seed_championship
```

## Деплой

Прод живёт на общем сервере (host-nginx фронтит несколько проектов — **порты 80/443 не
занимать**). Наши сервисы слушают только loopback, nginx проксирует по домену.

Эталон конфигов и пошаговый runbook — в [docs/deploy/](docs/deploy). Кратко:

```bash
# на сервере, каталог /opt/legalhunter
git pull --ff-only origin main
cd deployment
export RELEASE_SHA=$(cd /opt/legalhunter && git rev-parse HEAD) BUILD_TIME=$(date -u +%FT%TZ)
docker compose -f docker-compose.prod.yml --env-file prod.env up -d --build
curl -s https://legalhunter.pro/api/version   # release_sha должен совпасть, не "unknown"
```

Правила работы с прод-сервером и git-дисциплина многоагентной разработки — в
[CLAUDE.md](CLAUDE.md).

## Наследие (важно для контрибьютора)

Проект вырос из геймифицированного тренажёра, поэтому в коде остаются следы старой
архитектуры. Они **намеренно не удалены целиком**, потому что часть инфраструктуры
переиспользуется живым продуктом:

- **Тест-движок** обслуживается через роут `/pvp/quiz/[sessionId]` и `ws/knowledge.py` —
  несмотря на «pvp» в имени, это **одиночный тест по ФЗ-127**, а не мультиплеер.
- Сервисы `arena_*.py`, `glicko2.py`, модель `pvp.py` живы, потому что тест использует
  их для адаптивной сложности (`PvPRating`) — трогать хирургически, не сносить оптом.
- Роутеры `arena_lifelines.py` / `arena_powerups.py` определены, но **не подключены** —
  это мёртвый код, кандидат на удаление.

Чистка наследия ведётся аккуратно, отдельными выверенными проходами с проверкой через CI.
